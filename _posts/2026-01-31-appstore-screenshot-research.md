## 1. 스크린샷 구조 (Anatomy)

### 1.1 핵심 통계

| 지표 | 수치 |
|------|------|
| 사용자가 제품 페이지에서 보내는 평균 시간 | **7초** |
| 첫 1-3장 스크린샷 조회율 | **~100%** |
| 전체 스크린샷 스크롤하는 비율 | **~9%** |
| 세로 스크린샷 확대해서 보는 비율 | **~4%** |
| 앱 설명 "더 보기" 클릭 비율 | **~2%** |
| 첫 인상 형성 시간 (Google 연구) | **17~50밀리초** |

> **핵심:** 스크린샷이 전환의 거의 모든 무거운 짐을 지고 있다. 설명은 대부분 읽히지 않는다.

### 1.2 캡션 영역 vs 스크린샷 영역 비율

```
┌─────────────────────┐   ┌─────────────────────┐   ┌─────────────────────┐
│    📝 캡션 (25-30%)  │   │                     │   │                     │
│   "사진을 예술로     │   │   📱 풀 블리드       │   │   📱 스크린 (60%)    │
│    바꾸세요"         │   │   스크린샷 (100%)    │   │                     │
│─────────────────────│   │   (텍스트 오버레이)   │   │─────────────────────│
│                     │   │                     │   │    📝 캡션 (25-30%)  │
│   📱 스크린 (70%)    │   │                     │   │   하단 텍스트형       │
│                     │   │                     │   │                     │
└─────────────────────┘   └─────────────────────┘   └─────────────────────┘
   ① 상단 텍스트형            ② 풀 블리드형              ③ 하단 텍스트형
   (가장 일반적)              (몰입형 앱/게임)            (덜 사용됨)
```

**비율 가이드:**
- **상단 텍스트형 (가장 인기):** 캡션 25-30% + 스크린 70-75%
- **풀 블리드형:** 스크린 100% + 반투명 오버레이 텍스트
- **하단 텍스트형:** 스크린 60-70% + 하단 캡션 30-40%
- **50:50 분할형:** 캡션과 스크린이 동등 비율 (피처 하이라이트에 적합)

### 1.3 배경 처리 유형

| 유형 | 설명 | 적합한 앱 | 예시 |
|------|------|-----------|------|
| **단색** | 깨끗하고 모던한 느낌 | 생산성, 유틸리티 | Airbnb (흰색), Uber (검정) |
| **그라데이션** | 세련되고 역동적 | 소셜, 음악, 라이프스타일 | Spotify, Instagram |
| **패턴** | 브랜드 개성 강조 | 게임, 교육 | Duolingo |
| **사진/일러스트** | 감성적/맥락적 전달 | 여행, 음식, 피트니스 | Calm (자연 배경) |
| **다크 배경** | 프리미엄/몰입감 | 미디어, 게임, 프리미엄 앱 | Netflix, TikTok |

### 1.4 디바이스 프레임 유무

```
┌──────────────────┐   ┌──────────────────┐   ┌──────────────────┐
│  ┌────────────┐  │   │ ┌──────────────┐ │   │                  │
│  │            │  │   │ │              │ │   │  전체 스크린만    │
│  │ 📱 실제    │  │   │ │ 📱 프레임리스 │ │   │  (풀 블리드)     │
│  │  디바이스  │  │   │ │  (그림자만)   │ │   │                  │
│  │  프레임    │  │   │ │              │ │   │  프레임 없음      │
│  │            │  │   │ └──────────────┘ │   │                  │
│  └────────────┘  │   │                  │   │                  │
│                  │   │                  │   │                  │
└──────────────────┘   └──────────────────┘   └──────────────────┘
    ① 프레임 목업           ② 프레임리스           ③ 풀스크린
    (~50% 앱 사용)         (모던/미니멀)           (게임/몰입형)
```

**통계:** 상위 앱의 **~50%**가 실제 디바이스 목업 사용, 나머지는 프레임리스 또는 풀스크린

**주의:** 디바이스 프레임 사용 시 **최신 기기** 사용 필수 (iPhone 16 Pro Max 등). 구형 기기 프레임은 신뢰도를 떨어뜨림.

### 1.5 페이지 간 스토리텔링 흐름

**A.I.D.A. 프레임워크** (전환율 높은 구조):

```
스크린샷 1       스크린샷 2       스크린샷 3       스크린샷 4-6      스크린샷 7-10
┌───────────┐   ┌───────────┐   ┌───────────┐   ┌───────────┐    ┌───────────┐
│           │   │           │   │           │   │           │    │           │
│ ATTENTION │   │ INTEREST  │   │  DESIRE   │   │  DESIRE+  │    │  ACTION   │
│           │   │           │   │           │   │           │    │           │
│ 핵심 가치  │   │ 핵심 기능  │   │ 신뢰/증거 │   │ 추가 기능  │    │ CTA/마무리 │
│ 제안       │   │ 시연      │   │ 소셜프루프 │   │ 차별화    │    │           │
│           │   │           │   │           │   │           │    │           │
│ "더 나은   │   │ 실제 앱   │   │ ★★★★★    │   │ 고급 기능  │    │ 지금      │
│  수면을"   │   │ 화면      │   │ "100만+   │   │ 독특한    │    │ 시작하세요 │
│           │   │           │   │  사용자"  │   │ 기능들    │    │           │
└───────────┘   └───────────┘   └───────────┘   └───────────┘    └───────────┘
```

**Value-Usage-Trust 프레임워크** (실무에서 더 간결):

| 순서 | 역할 | 설명 | 예시 |
|------|------|------|------|
| **#1** | 💎 가치 약속 (Value Promise) | 가장 중요한 이점을 즉시 전달 | Calm: "Sleep better, feel better" |
| **#2** | 🔧 핵심 사용법 (Key Usage) | 앱이 어떻게 작동하는지 한 가지 명확한 액션으로 보여줌 | Duolingo: 실제 레슨 화면 |
| **#3** | 🏆 신뢰/소셜 프루프 (Trust) | 평점, 수상, 미디어 인용, 다운로드 수 | Klarna: Trustpilot 평점 표시 |
| **#4-10** | ✨ 부가 기능/차별화 | 2차 기능, 유니크한 차별점, 추가 사용 사례 | 고급 기능, 커스터마이징 등 |

## 2. 레이아웃 패턴 12가지

### 패턴 1: 센터 디바이스 + 상단 텍스트 (가장 인기)

```
┌─────────────────────┐
│                     │
│   "할 일을 쉽게     │
│    관리하세요"       │
│                     │
│    ┌───────────┐    │
│    │           │    │
│    │  📱 앱 화면 │    │
│    │           │    │
│    │           │    │
│    └───────────┘    │
│                     │
└─────────────────────┘
```

- **장점:** 깔끔, 읽기 쉬움, 가장 검증된 패턴
- **단점:** 차별화 어려움 (너무 많은 앱이 사용)
- **적합:** 생산성, 유틸리티, 금융, 비즈니스
- **예시:** Airbnb, Spotify, Revolut, Notion

### 패턴 2: 듀얼 디바이스 (비교/전후)

```
┌─────────────────────┐
│   "전/후 비교"       │
│                     │
│  ┌────────┐ ┌────────┐
│  │        │ │        │
│  │ Before │ │ After  │
│  │        │ │        │
│  │        │ │        │
│  └────────┘ └────────┘
│                     │
└─────────────────────┘
```

- **장점:** 기능 대비를 직관적으로 보여줌
- **단점:** 각 화면이 작아져 세부사항 파악 어려움
- **적합:** 사진 편집, 뷰티, 헬스/피트니스 (전후 비교)
- **예시:** Facetune, VSCO

### 패턴 3: 앵글드 디바이스 (기울어진 목업)

```
┌─────────────────────┐
│   "새로운 경험"      │
│                     │
│      ╱─────────╲    │
│     ╱           ╲   │
│    │  📱 기울어진 │   │
│    │    목업      │   │
│     ╲           ╱   │
│      ╲─────────╱    │
│                     │
└─────────────────────┘
```

- **장점:** 시각적으로 역동적, 현대적, 시선 집중
- **단점:** 화면 내용 읽기 어려울 수 있음
- **적합:** 라이프스타일, 소셜, 게임, 미디어
- **예시:** TripAdvisor, Instagram

### 패턴 4: 풀 블리드 (프레임 없이 스크린샷만)

```
┌─────────────────────┐
│█████████████████████│
│█████████████████████│
│███ 앱 UI 전체 화면 ███│
│███ (프레임 없음)   ███│
│█████████████████████│
│█████████████████████│
│█████████████████████│
│█████████████████████│
└─────────────────────┘
```

- **장점:** 최대 화면 공간 활용, 몰입감 극대화
- **단점:** 캡션 공간 부족, 기능 설명 제한적
- **적합:** 게임, 사진/카메라, 미디어 플레이어
- **예시:** TikTok, Netflix, 대부분의 게임

### 패턴 5: 피처 하이라이트 (아이콘 + 설명)

```
┌─────────────────────┐
│                     │
│   🔔  푸시 알림      │
│   ─────────────     │
│   실시간 업데이트를   │
│   놓치지 마세요      │
│                     │
│    ┌───────────┐    │
│    │  관련 화면  │    │
│    └───────────┘    │
│                     │
└─────────────────────┘
```

- **장점:** 특정 기능을 명확하게 강조
- **단점:** 여러 기능을 한번에 보여주기 어려움
- **적합:** SaaS, 복잡한 기능의 앱, B2B
- **예시:** Slack, Asana, Notion

### 패턴 6: 파노라마/연결형 (여러 장이 이어지는)

```
┌──────────┐┌──────────┐┌──────────┐
│          ││          ││          │
│   화면 1  ││   화면 2  ││   화면 3  │
│   ───────>│>──────────│>─────    │
│   연결되는 ││  시각적   ││  스토리   │
│   배경     ││   흐름    ││          │
│          ││          ││          │
└──────────┘└──────────┘└──────────┘
```

- **장점:** 스크롤 유도, 스토리텔링 강화
- **단점:** 제작 복잡, 스크롤을 유도한다는 데이터 근거 부족
- **적합:** 브랜드 스토리가 강한 앱, 프리미엄 앱
- **예시:** Klarna, 일부 게임

### 패턴 7: 미니멀/텍스트 중심

```
┌─────────────────────┐
│                     │
│                     │
│   "당신의 마음을     │
│    돌보세요"         │
│                     │
│   ○ ○ ○ (미니멀     │
│          그래픽)    │
│                     │
│                     │
└─────────────────────┘
```

- **장점:** 감성적, 프리미엄 느낌, 차별화
- **단점:** 앱 실제 화면을 안 보여줌 (Apple 가이드라인 주의)
- **적합:** 명상, 웰니스, 프리미엄 라이프스타일
- **예시:** Headspace, Calm

### 패턴 8: 라이프스타일/인물 중심

```
┌─────────────────────┐
│                     │
│   👤 실제 사용자     │
│      사진 배경       │
│                     │
│    ┌───────────┐    │
│    │  📱 앱 화면 │    │
│    │  (오버레이) │    │
│    └───────────┘    │
│                     │
│   "당신만을 위한"    │
└─────────────────────┘
```

- **장점:** 감정적 연결, 타겟 오디언스 공감
- **단점:** Apple은 사람이 기기를 들고 있는 이미지 금지
- **적합:** 데이팅, 소셜, 피트니스 (주의: Apple 규정 준수 필요)
- **예시:** Bumble, Tinder

### 패턴 9: 소셜 프루프/수상 중심

```
┌─────────────────────┐
│                     │
│   🏆 "2025년        │
│   올해의 앱"         │
│                     │
│   ★★★★★ 4.9         │
│   "100만+ 다운로드"  │
│                     │
│   📰 "TechCrunch,   │
│   Forbes 선정"       │
│                     │
└─────────────────────┘
```

- **장점:** 즉각적 신뢰 구축
- **단점:** Apple은 검증 불가 주장 금지 (주의)
- **적합:** 성숙한 앱, 수상 경력 있는 앱 (3번째 스크린샷으로 권장)
- **예시:** Strava (언론 인용), Asana (수상)

### 패턴 10: Gen-Z/바이브 중심

```
┌─────────────────────┐
│  ✨ 🎈 🌈            │
│                     │
│  "finally...        │
│   a map that shows  │
│   places you        │
│   actually want     │
│   to go to"         │
│                     │
│  ┌───────────┐      │
│  │ 3D 그래픽  │ 🎈   │
│  └───────────┘      │
└─────────────────────┘
```

- **장점:** 타겟 오디언스와 강한 감정적 연결
- **단점:** 범용성 떨어짐, 특정 세대에만 어필
- **적합:** Gen-Z 타겟 소셜/라이프스타일 앱
- **예시:** Corner, BeReal

### 패턴 11: 다크 모드/프리미엄

```
┌─────────────────────┐
│█████████████████████│
│█████████████████████│
│███  흰색 텍스트   ███│
│███  "프리미엄     ███│
│███   경험"        ███│
│███ ┌───────────┐ ███│
│███ │ 다크 UI   │ ███│
│███ └───────────┘ ███│
│█████████████████████│
└─────────────────────┘
```

- **장점:** 시각적 차별화 (대부분 라이트 모드), 프리미엄 느낌
- **단점:** 텍스트 가독성 주의 필요
- **적합:** 미디어, 프리미엄 앱, 개발자 도구
- **예시:** Uber (다크 배경 + 흰색 폰트)

### 패턴 12: "Ugly Ads" / 진정성 중심 (2025 신규 트렌드)

```
┌─────────────────────┐
│                     │
│  📱 실제 사용자처럼  │
│  보이는 스크린샷     │
│                     │
│  꾸밈없는 실제 화면  │
│  캐주얼한 텍스트     │
│                     │
│  "진짜 사용자        │
│   스크린샷 같은 느낌" │
│                     │
└─────────────────────┘
```

**핵심 규칙:**

3. **2-3개 주요 색상** — 갤러리 전체 일관성 유지

- 앱이 다크 모드를 지원한다면, **최소 1장은 다크 모드로** 보여줄 것을 권장

### 5.2 첫 번째 스크린샷의 중요성

> **"Nearly 100% of users view the first 1-3 screenshots when visiting a product page, but only 9% scroll through all screenshots."** — Picc.co

- 사용자는 **7초 이내** 다운로드 여부 결정

- **60%**의 사용자가 **5-7초 이내** 설치 결정

| 적합한 앱 | 시각적 앱 (게임, 디자인, 에디터) | 모든 앱 |
| 제작 비용 | 높음 | 낮음 |
| Apple 요구 | 15-30초, 실제 앱 화면만 | 필수 |
| 위치 | 첫 번째 위치에 표시 | 비디오 뒤에 표시 |

**권장:** 앱이 시각적으로 매력적이면 비디오 추가, 유틸리티/생산성 앱은 강력한 스크린샷이 동등하거나 더 효과적

### 5.4 A/B 테스트 플랫폼

| 플랫폼 | 특징 |
|--------|------|
| **Apple PPO** | 최대 3개 변형 + 원본, 90일 최대, 아이콘/스크린샷/비디오 테스트 |
| **Google Play Experiments** | 무제한 기간, 설명까지 테스트 가능, 5개 로컬 실험 동시 |
| **SplitMetrics** | 출시 전 테스트, 제어된 환경에서 사용자 행동 시뮬레이션 |
| **AppTweak** | 경쟁사 분석, 트렌드 벤치마크 |

### 5.5 테스트 전략

1. **한 번에 하나의 변수만** 테스트 — 여러 변경 동시 테스트 시 무엇이 효과인지 알 수 없음
2. **첫 번째 스크린샷부터** 테스트 — 가장 높은 영향력
3. **연 2-4회** 스크린샷 업데이트 (상위 200개 앱 기준)
4. **상위 게임:** Google Play에서 연 최대 8회 비주얼 업데이트
5. 각 업데이트 후 **전환율 모니터링** 필수

## 7. 실전 체크리스트

### 7.1 제작 전 체크리스트

- [ ] 타겟 오디언스 정의 완료
- [ ] 경쟁 앱 스크린샷 분석 완료 (최소 5개)
- [ ] 핵심 가치 제안(Value Proposition) 1문장으로 정리
- [ ] A.I.D.A. 또는 Value-Usage-Trust 프레임워크 적용 결정
- [ ] 레이아웃 패턴 선택 (12가지 중)
- [ ] 색상 팔레트 결정 (브랜드 일관성)
- [ ] 캡션 문구 초안 작성 (공식 적용)

### 7.2 디자인 체크리스트

- [ ] 첫 3장 스크린샷에 가장 강력한 메시지 배치
- [ ] 캡션 3-7 단어 (영어) / 5-15자 (한국어) 이내
- [ ] 폰트: Bold 이상, 산세리프, 고대비
- [ ] 실제 기기 썸네일 크기에서 가독성 확인
- [ ] 디바이스 프레임 사용 시 최신 기기 (iPhone 16 Pro Max)
- [ ] 상태 바(배터리, 시간, 와이파이) 정리
- [ ] 안전 영역(Safe Area) 내에 콘텐츠 배치

### 7.3 기술 체크리스트

- [ ] iPhone: 1290 × 2796 px (6.9") — 필수
- [ ] iPad: 2064 × 2752 px (13") — iPad 지원 시 필수
- [ ] 파일 형식: PNG (추천) 또는 JPEG
- [ ] 색공간: RGB (CMYK ❌)
- [ ] 투명도: 없음 (알파 채널 ❌)
- [ ] 파일 크기: 10MB 이하/장
- [ ] 100% 줌에서 최종 확인

### 7.4 콘텐츠 체크리스트

- [ ] 실제 앱 UI만 포함 (가짜 UI, 컨셉 디자인 ❌)
- [ ] 검증 불가 주장 없음 ("#1", "최고" ❌)
- [ ] 공격적 CTA 없음 ("지금 다운로드" ❌)
- [ ] 다른 플랫폼 언급 없음
- [ ] 개인정보 없음 (플레이스홀더/익명화 사용)
- [ ] 저작권 침해 없음
- [ ] 현재 앱 버전을 정확히 반영

### 7.5 출시 후 체크리스트

- [ ] 전환율 모니터링 시작
- [ ] 2주 후 데이터 분석
- [ ] A/B 테스트 계획 수립 (첫 스크린샷부터)
- [ ] 분기별 스크린샷 업데이트 일정 수립
- [ ] 로컬라이제이션 우선 시장 결정
