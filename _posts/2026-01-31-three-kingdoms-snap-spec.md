## 3. 핵심 메카닉

### 3.1 게임 흐름
1. 12장 덱으로 시작, 3장 초기 핸드, 매 턴 1장 드로우 (총 9장 접근)
2. 6턴 진행, 턴 번호 = 사용 가능 군량
3. 3개 전장에 동시에 카드 배치 (전장당 최대 4장)
4. 양측 턴 종료 후 동시 공개 (승리 중인 측 먼저)
5. 2/3 전장 승리 시 게임 승리

### 3.2 군량(에너지) 시스템
- 1턴 = 1군량, 2턴 = 2군량, ..., 6턴 = 6군량
- 미사용 군량 이월 없음
- 카드 코스트 ≤ 현재 군량이면 배치 가능
- 한 턴에 여러 카드 배치 가능 (군량 허용 범위 내)

### 3.3 전장(Location) 시스템
- 3개 전장: 좌(1턴 공개), 중(2턴 공개), 우(3턴 공개)
- 미공개 전장에도 카드 배치 가능
- 각 전장 고유 효과 (15개 풀에서 랜덤 선택)
- 전장당 각 측 최대 4장

### 3.4 카드 능력
- **출전 효과 (On Reveal):** 카드 공개 시 1회 발동
- **지속 효과 (Ongoing):** 카드가 장에 있는 동안 매 계산마다 적용
- **없음 (Vanilla):** 능력 대신 높은 기본 파워

### 3.5 결전(SNAP) & 퇴각(Retreat)
- 기본 전공: 1
- 결전 선언 시 전공 2배 (양측 각 1회)
- 게임 종료 시 자동 2배
- 최대 전공: 8 (양측 결전 + 게임 종료)
- 퇴각 시 현재 턴 시작 시점의 전공만큼 패배

### 3.6 승패 판정
1. 각 전장에서 총 파워 비교
2. 2개 이상 전장 승리 시 게임 승리
3. 전장 동점 시: 전체 총 파워 비교
4. 전체 동점: 무승부

## 6. AI 전략

### 6.1 AI 덱 (4종 랜덤 선택)
1. **위 제어덱**: 조조, 사마의, 장료, 하후돈, 순욱, 곽가, 전위, 허저, 조인, 여포, 화타, 동탁
2. **촉 결속덱**: 유비, 관우, 장비, 제갈량, 조운, 마초, 황충, 방통, 관흥, 화타, 여포, 초선
3. **오 파괴덱**: 손권, 주유, 여몽, 감녕, 육손, 태사자, 황개, 손책, 대교소교, 노숙, 동탁, 장각
4. **혼합 공격덱**: 조조, 관우, 주유, 여포, 전위, 태사자, 곽가, 감녕, 화타, 장각, 허저, 마초

### 6.2 AI 의사결정
1. 에너지 내에서 플레이 가능한 카드 조합 탐색
2. 각 전장 승리 가능성 평가
3. 높은 코스트 카드 우선, 전장 분산 배치
4. 결전: 2개 전장 이상 우세 시 결전 선언
5. 퇴각: 3개 전장 모두 열세 시 확률적 퇴각

## 7. UI/UX 디자인

### 7.1 비주얼 테마
- **수묵화/한지 느낌**: 다크 파치먼트 배경, 한지 텍스처
- **색상**: 붉은색 + 금색 테마, 먹색 텍스트
- **세력 색상**: 위=파랑, 촉=초록, 오=빨강, 무=회색
- **카드 디자인**: CSS로 구현 (이모지 아이콘, 서예체 이름)

### 7.2 화면 구성
1. **메뉴**: 게임 시작, 진용 편성, 전적, 도움말
2. **덱빌더**: 36장 중 12장 선택
3. **게임**: 3전장 보드 + 핸드 + 액션 버튼
4. **결과**: 승/패/무 연출 + 전공 획득

### 7.3 인터랙션
- 카드 탭 → 선택 → 전장 탭 → 배치 (모바일 최적화)
- 드래그앤드롭 지원 (데스크톱)
- 카드 길게 누르기 → 상세 정보

## 8. 기술 스택
- 단일 HTML5 파일 (CSS + JS 인라인)
- TG SDK + tg-sdk-wrapper
- PWA (manifest, service worker 등록)
- localStorage (덱, 전적, 설정)
- Web Audio API (UI 사운드)
- CSS 3D Transform (카드 애니메이션)
- i18n (한국어/영어)
