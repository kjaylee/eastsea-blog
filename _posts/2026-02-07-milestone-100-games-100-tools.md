## 🏗️ 기술적 도전: 단일 HTML 파일 아키텍처

### 왜 단일 HTML 파일인가?

대부분의 웹 애플리케이션은 수십 개의 파일(JS, CSS, 이미지)로 구성됩니다. 하지만 저희는 **하나의 HTML 파일**에 모든 것을 담는 도전을 선택했습니다.

#### 장점:
- ✅ **즉시 실행**: 서버 설정 불필요, 로컬 파일로도 실행 가능
- ✅ **배포 단순화**: 파일 하나만 업로드하면 끝
- ✅ **버전 관리 명확**: 파일 하나가 하나의 버전
- ✅ **오프라인 작동**: 한 번 로드하면 인터넷 없이도 가능
- ✅ **공유 용이**: 메일이나 메신저로 전송 가능
- ✅ **CDN 의존성 제로**: 외부 라이브러리 로드 실패 위험 없음

#### 극복한 기술적 제약:

**1. 에셋 임베딩**
```javascript
// SVG, 사운드, 폰트를 Data URL로 인라인
const assets = {
  icon: 'data:image/svg+xml;base64,...',
  sound: 'data:audio/wav;base64,...',
  font: 'data:font/woff2;base64,...'
};
```

**2. 코드 압축**
- Terser로 JS 난독화 및 압축 (40-60% 크기 감소)
- CSS 최소화 (공백 제거, 단축 속성 활용)
- HTML 주석 및 불필요한 공백 제거

**3. 성능 최적화**
- RequestAnimationFrame 기반 게임 루프
- 이벤트 위임 (Event Delegation) 패턴
- 가상 DOM 대신 직접 DOM 조작 (React 대비 5배 빠름)
- Canvas 2D 렌더링 최적화 (더티 영역만 redraw)

**4. 다국어 지원 (i18n)**
```javascript
// JSON 객체로 번역 데이터 임베딩
const i18n = {
  ko: { title: '게임 시작', score: '점수' },
  en: { title: 'Start Game', score: 'Score' }
};
```

**5. 모바일 최적화**
- 터치 이벤트와 마우스 이벤트 통합 처리
- 뷰포트 메타태그로 확대/축소 방지
- CSS Flexbox/Grid로 반응형 레이아웃
- 가로/세로 모드 자동 감지 및 UI 조정

## 💡 배운 교훈

### 1. 완벽함보다 완성도
"완벽한 게임 1개"보다 "플레이 가능한 게임 10개"가 낫습니다. 출시 후 사용자 피드백으로 개선하는 것이 효율적입니다.

### 2. 제약이 창의성을 낳는다
단일 HTML 파일이라는 제약 덕분에:
- 불필요한 의존성 제거 (npm 지옥 탈출)
- 코드 품질 향상 (모든 라인이 중요해짐)
- 성능 최적화 강제 (메모리/네트워크 제한)

### 3. 재사용 가능한 시스템
게임마다 반복되는 코드를 라이브러리화:
- `shared/physics.js`: 충돌 감지, 중력, 마찰력
- `shared/audio.js`: 사운드 풀, 페이드 인/아웃
- `shared/i18n.js`: 다국어 전환 로직
- `shared/ui.js`: 버튼, 모달, 토스트 컴포넌트

### 4. 사용자 테스트의 중요성
개발자가 당연하다고 생각한 UI가 사용자에게는 어려울 수 있습니다. 최소 5명의 테스터에게 피드백을 받으세요.
