- **9. Cursor 3, エージェント管理コンソール로 전면 재설계 — Composer 2 내재화**

Cursor(Anysphere Inc.)가 4월 2일 Cursor 3를 출시하며 코드 에디터 개념 자체를 탈바꿈시켰다. XenoSpectrum 리포트에 따르면 Cursor 3는 VS Code 포크를 넘어 에이전트 중심의 새로운 UI를 零から設計했다. 最大 변화는 사이드바 구조로, 로컬·클라우드 에이전트가 모두 단일视图에 통합된다. 클라우드 에이전트는 자동 데모 영상·스크린샷을 생성해 개발자가 작업 현황을 즉시 검증할 수 있다. 세션의 클라우드-로컬 간 동적 이동 기능도 도입됐다 — 무거운 작업을 클라우드로 옮기고 로컬에서 확인·수정하는 흐름이 자유로워졌다. 이를 견인하는 내재 모델이 **Composer 2**로, Cursor가 자체 개발한 코딩 특화 LLM이다. Claude 등 범용 LLM 대비 사용량上限 높고 반복 작업速度快며 비용 효율성도 우수.

- **10. GitHub Copilot CLI, Rubber Duck 모드 도입 — 75% 성능 갭 해소**

GitHub가 Copilot CLI에 실험적新기능 "Rubber Duck" 모드를 도입했다. Publickey.jp报道에 따르면 이 모드는 메인 AI 모델과는 다른 계열의 AI를 세컨드 오피니언 검토자로 활용한다. 예컨대 Claude Sonnet을 메인으로 선택하면 GPT-5.4가 자동으로 리뷰어로 배치된다. GitHub 자사 평가에서 Claude Sonnet + Rubber Duck은 Sonnet 단독 대비 성능 갭의 **74.7%**를 메웠다고 밝혔다. 3개 이상 파일에 걸친 문제나 70스텝 이상의 복잡한 작업에서 특히 효과적. Rubber Duck이라는 이름은 프로그래머가 고무 오리에게 코드를 설명하며 디버깅하는 전설적 "라버덕/debugging"에서 유래했다. AI가 AI를 리뷰하는 구조로, 인간 개발자의 역할이 코칭으로 전환되는有趣現象.

## 💡 Miss Kim의 인사이트
