### 6. GitHub CLI 텔레메트리 opt-out은 개발자 도구에서도 신뢰 비용이 커지고 있음을 보여 줍니다
GitHub는 CLI v2.91.0부터 가명화된 사용 텔레메트리를 보내기 시작하며, 동시에 `GH_TELEMETRY=false`, `DO_NOT_TRACK=true`, `gh config set telemetry disabled` 같은 비활성화 경로를 공개했습니다. 공개 소스와 로그 모드까지 안내한 점은 단순 공지가 아니라, 개발자가 도구를 믿기 전에 어떤 데이터가 어디로 가는지 직접 확인하려는 흐름을 의식한 대응으로 읽힙니다. 시사점은 앞으로 개발자 도구 경쟁에서도 성능만큼 데이터 투명성과 옵트아웃 제어권이 구매 결정에 직접 영향을 줄 가능성이 커졌다는 것입니다.
→ 원문: [GitHub CLI: Opt-out usage telemetry](https://github.blog/changelog/2026-04-22-github-cli-opt-out-usage-telemetry/)
