# Game Workspace Map (2026-04-21)

## 결론
게임 관련 파일은 실제로 여러 곳에 흩어져 있었고, 이제 판단 기준을 아래처럼 고정한다.

1. **현재 배포 자산 기준**: `games/`
2. **전체 카탈로그 기준**: `docs/game-catalog-full-2026-04-21.md`, `docs/game-catalog-full-2026-04-21.csv`
3. **원본/백로그 창고**: `../games/` (워크스페이스 루트)
4. **기획/실험 문서**: `../specs/`
5. **중복 정리 흔적**: `../archive/game-dedup-2026-04-19/`, `../artifacts/game-dedup/`

## 왜 헷갈렸는가
- `workspace/games/` 와 `eastsea-blog/games/` 가 동시에 존재했다.
- `specs/`, `.state/`, `archive/`, `artifacts/` 에도 게임 관련 폴더가 많이 남아 있었다.
- 과거 배치/실험 흔적이 정리되지 않아 실제 배포 기준과 원본 창고 기준이 섞여 보였다.

## 운영 규칙
- 게임을 찾을 때는 먼저 `games/` 를 본다.
- 게임 목록이나 링크를 찾을 때는 카탈로그 문서를 본다.
- 루트 `../games/` 는 배포 판단 기준이 아니라 소스/백로그 창고로 취급한다.
- 실제 이동/삭제는 비파괴 검증 후 별도 수행한다.
