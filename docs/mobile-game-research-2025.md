```
- HTML5/Godot Web 빌드

        ↓ (성공 시)

[2단계: Google Play + App Store]
- Godot 네이티브 빌드 (Android/iOS)
- 수익화 강화 (광고 SDK, IAP)
- ASO 최적화
- 비용: Google Play $25 + App Store $99/년
- 목표: 매출 + 스케일

        ↓ (성공 시)

[3단계: Steam + 웹]
- Godot PC 빌드 + 웹 포털 (itch.io, CrazyGames)
- 프리미엄 또는 확장 버전
- 비용: Steam $100/앱
- 목표: 추가 수익 + 브랜드

        ↓ (성공 시)

[4단계: 콘솔 (선택)]
- Godot 콘솔 빌드 (Switch, Xbox)
- 퍼블리셔 파트너십
- 목표: 최대 도달
```

## 🔑 10. 핵심 시사점 요약

### 시장이 말하는 것
1. **심플한 퍼즐이 왕이다:** Block Blast (356M DL, $1M/일) — 복잡할 필요 없음
2. **하이브리드가 미래다:** 순수 하이퍼캐주얼은 죽고, 메타+수익화 레이어가 필수
3. **라이브옵스가 생명이다:** 한 번 출시하고 끝이 아닌, 지속적 이벤트/업데이트
4. **광고는 보수적으로:** Block Blast의 핵심은 "player-first" 광고 정책
5. **AI/A/B 테스트가 차별화:** 성공 게임들은 모두 데이터 기반 의사결정
6. **인디는 독창성으로 승부:** 기존 장르의 창의적 결합이 핵심
7. **PC 퍼스트 → 모바일은 유효한 전략:** Balatro, Vampire Survivors 사례
8. **iOS 유저가 더 많이 쓴다:** iOS 우선 출시 또는 동시 출시 권장

### J&J Games에 적용할 점
- **Godot 4.6으로 충분하다:** 하이브리드캐주얼, 2D 퍼즐, 로그라이크 모두 가능
- **텔레그램 Mini App은 최고의 테스트베드:** 무료 유통 + 바이럴 + 빠른 피드백
- **블록/소트/스크류 퍼즐부터:** 개발 비용 낮고 시장 급성장
- **하이브리드 수익화 필수:** 광고만으론 부족, IAP 레이어 추가
- **라이브옵스 설계:** 출시 전부터 이벤트/시즌 시스템 기획 필요
