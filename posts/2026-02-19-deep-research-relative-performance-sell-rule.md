# 인덱스 대비 초과수익률 기반 매도 경고 룰 딥리서치

- 작성일: 2026-02-19
- 대상 기본 룰
  - \(E_t = R^{stock}_t - R^{index}_t\) (초과수익률)
  - \(H_t = \max_{0\le i\le t} E_i\)
  - 경고 조건: \(H_t - E_t \ge 10\%p\)

---

## 0) 한 줄 정의
이 룰은 **"종목의 벤치마크 대비 성과(알파) 곡선의 고점 대비 하락폭"**을 감지하는 **상대성과 기반 트레일링 스톱(상대 드로우다운) 경고**다.

---

## 1) 수학적 정의와 논리 일관성 검증

## 1.1 엄밀한 정의(권장)
실무 혼선을 줄이려면 먼저 반환값 정의를 고정해야 한다.

- 기준시점(포지션 시작/리밸런싱 기준일) \(t_0\)
- 총수익률(배당·분할 반영) 기반 누적수익률:
  - \(R^{stock}_t = \frac{TR^{stock}_t}{TR^{stock}_{t_0}} - 1\)
  - \(R^{index}_t = \frac{TR^{index}_t}{TR^{index}_{t_0}} - 1\)
- 초과수익률:
  - \(E_t = R^{stock}_t - R^{index}_t\)
- 상대 고점 대비 하락폭:
  - \(D_t = H_t - E_t\), \(H_t = \max(E_{t_0},...,E_t)\)
- 경고:
  - \(D_t \ge \delta\), 기본 \(\delta=10\%p\)

> 주의: 일별 수익률 차이 \(r^{stock}_t-r^{index}_t\)를 단순 누적한 값과 위 누적수익률 차이는 엄밀히 동일하지 않다(복리 효과). 구현에서 하나로 통일해야 한다.

## 1.2 논리 일관성
- 내부 모순은 없다. \(H_t\)는 비감소, \(D_t\ge0\) 보장.
- 조건은 **절대손실 여부와 독립**이다.
  - 예: 종목 +12%, 지수 +25%이면 \(E_t=-13%p\), 경고 가능.
- 따라서 이 룰은 "손절"이 아니라 "**알파 붕괴 경고**" 성격이다.

## 1.3 해석상 필수 합의 포인트(미합의 시 모순처럼 보일 수 있음)
1. 수익률 단위: 단순수익률 vs 로그수익률
2. 가격 종류: 종가 vs 조정종가(배당/분할 반영)
3. 비교 지수: 가격지수 vs 총수익지수(TR)
4. 통화: 종목·지수를 동일 통화로 환산할지 여부
5. 기준시점 재설정: 신규/추가매수 시 \(t_0,H_t\)를 유지·리셋·가중 병합 중 무엇으로 할지

## 2) 핵심 케이스 검증

## 2.1 시장 국면별

### (A) 상승장
- 종목과 지수가 동반 상승해도 지수가 더 강하면 \(E_t\) 하락.
- 경고는 "수익 중"에도 발생 가능(절대손실과 불일치).
- 해석: 수익 실현보다 **대안비용(기회비용) 증가** 경고.

### (B) 하락장
- 지수 급락보다 종목이 덜 빠지면 \(E_t\) 상승(방어력 우위).
- 이후 반등장에서 종목 반등이 약하면 \(E_t\) 급락해 경고 가능.
- 즉 하락장 방어주가 반등장에서 알파를 반납할 때 민감.

### (C) 횡보장
- 잡음(노이즈)로 \(E_t\) 작은 진동 반복.
- 임계치가 고정(10%p)면 보통 둔감하지만, 단기 급등/급락 종목은 오탐 가능.

## 2.2 종목 지속 언더퍼폼
- 시작부터 계속 지수보다 약하면 \(E_t\)가 단조 하락.
- \(E_{t_0}=0\)이면 \(H_t=0\), 경고 시점은 \(E_t\le-10%p\).
- 장점: 명확. 단점: 초반 구조적 약세 탐지까지 시간 지연 가능.

## 2.3 인덱스 급락 후 반등, 종목 갭/급등락
- 지수 급락 구간에서 \(E_t\)가 높아졌다가(방어) 반등기에 되돌림이 크면 빠른 경고.
- 종목 단일 갭다운/악재 하루로 \(D_t\) 임계 초과 가능(1일 노이즈 경고).
- 보완: "지속일수 N일" 또는 "종가기준 2회 확인" 필요.

## 2.4 신규매수·분할매수·추가매수(평단 변경)
이 룰의 실무 핵심 난점.

- **신규매수 직후**: 데이터 충분치 않아 초기 \(H_t\)가 0 근처. 즉시 경고를 피하려면 워밍업 기간 필요.
- **분할매수/추가매수** 처리 옵션
  1. **Lot 분리 방식(권장 정확도 높음)**: 매수 lot마다 \(E,H\) 독립 추적 후 비중 가중 경고.
  2. **포지션 리베이스 방식(단순)**: 추가매수일에 \(t_0\) 재설정, \(E=0,H=0\). 과거 정보 소실.
  3. **연속 방식(중간)**: \(H\) 유지 + \(E\)만 자본가중 재계산. 구현 복잡, 해석 난이도 높음.

## 2.5 배당/분할/병합/거래정지/상폐/서킷브레이커
- 배당: 종목은 TR/조정종가 필수. 지수도 가능하면 TR지수 사용.
- 분할/병합: 미조정 가격 사용 시 허위 급락·급등으로 오경고 발생.
- 거래정지: 가격 고정 기간 \(E_t\) 왜곡 가능(지수만 움직임).
- 상폐: 최종청산 이벤트로 별도 "강제 종료" 규칙 필요(이 룰과 별개).
- 서킷브레이커/VI: 장중 급변보다 EOD 기준 사용 권장(체결잡음 완화).

## 2.6 데이터 결측/휴장 비동기/통화 차이(해외주식)
- 결측값: 무조건 전일값 보간 금지(허위 무변동).
- 휴장 비동기: 공통 거래일 캘린더 기준 정렬 필요.
- 통화차이: 종목 USD, 지수 KRW면 FX 요인이 알파에 혼입.
  - 해결 A: 둘 다 동일 통화 환산
  - 해결 B: 종목 상장국 지수로 비교(통화 자연 일치)

---

## 3) 오탐/미탐 시나리오

## 3.1 오탐(경고했지만 실제 매도 비효율)
1. 단기 이벤트성 갭다운 후 즉시 회복
2. 고베타 성장주의 정상 변동성 구간
3. 지수 급반등(매크로 숏커버)으로 상대성과 일시 악화
4. 추가매수 직후 기준 리셋 미흡으로 인위적 드로우다운 확대
5. 배당락/데이터 조정 지연으로 허위 \(E_t\) 하락

## 3.2 미탐(경고 못했지만 리스크 확대)
1. 절대손실이 큰데 지수도 같이 급락해 \(E_t\)는 안정
2. 저점부터 계속 약하지만 \(H_t\)가 낮아 임계 미도달(활성화 임계 도입 시 악화 가능)
3. 펀더멘털 훼손(실적 붕괴)이 상대성과 지표에 지연 반영
4. 장중 급락 후 종가 회복(종가 기준만 쓰면 장중 리스크 누락)

---

## 4) 실전 보완안

## 4.1 활성화 임계(Arming Threshold)
- 예: \(H_t \ge +5%p\)일 때만 룰 활성화.
- 장점: "성과를 확보한 뒤 반납"에 집중.
- 단점: 시작부터 언더퍼폼 종목 탐지 약화.
- 절충: 이중모드
  - Mode A(알파보호): \(H_t\ge+5\) 필요
  - Mode B(열위차단): \(E_t\le-10\) 별도 경고

## 4.2 지속일수 확인
- 조건: \(D_t\ge\delta\)가 연속 3거래일 유지 시 확정 경고.
- 장점: 1일 잡음 오탐 감소.
- 대안: 3일 중 2일 충족, 혹은 주봉 확인.

## 4.3 절대손실/추세/실적과 결합
- 절대손실 필터: 종목 누적수익률 \(R^{stock}_t\le-8\%\) 동시 충족 시 강경고.
- 추세 필터: 종가 < 120일선(또는 200일선)일 때 경고 강도 상향.
- 펀더멘털 필터: 실적 하향 가이던스/이익추정 하향 동반 시 즉시 검토.

## 4.4 변동성 기반 임계값
고정 10%p 대신 시장·종목 특성 반영:
- \(\delta_t = \max(\delta_{min}, k\cdot\sigma_E)\)
  - \(\sigma_E\): 최근 60일 \(E\) 변화율 표준편차
  - 예: \(\delta_{min}=6\%p, k=2.0\)
- 또는 상대비율 \(Q_t=P^{stock}_t/P^{index}_t\)의 ATR 기반 밴드.
- 권장: Floor/Ceiling 동시 적용(예: 6~15%p)으로 과민/둔감 방지.

## 5) 구현 스펙 초안

## 5.1 입력(Inputs)
1. 일자별 시계열
   - 종목 TR 또는 조정종가
   - 벤치마크 TR 또는 조정지수
   - (필요 시) FX 환율
2. 거래 원장
   - 매수/매도 일자, 수량, 단가, 수수료
3. 기업행위 이벤트
   - 배당, 분할/병합, 거래정지, 상폐
4. 파라미터
   - \(\delta\), arming threshold, persistence days, warmup days

## 5.2 상태변수(State)
- `base_date` (기준일)
- `E_t`, `H_t`, `D_t`
- `armed` (활성화 여부)
- `breach_count` (연속 충족일)
- `alert_state` (`NONE`/`WATCH`/`TRIGGERED`)
- (분할매수 시) lot별 `E,H,D,weight`

## 5.3 일별 업데이트 의사코드
```python
for t in trading_days:
    # 0) 데이터 검증
    if missing(stock[t]) or missing(index[t]):
        mark_quality_issue(t)
        continue  # 혹은 보수적으로 alert_state 유지

    # 1) 공통 기준 수익률 계산 (동일 통화, TR 기준 권장)
    R_stock = TR_stock[t] / TR_stock[base_date] - 1
    R_index = TR_index[t] / TR_index[base_date] - 1
    E = R_stock - R_index

    # 2) 고점/드로우다운 업데이트
    H = max(H_prev, E)
    D = H - E

    # 3) 활성화 조건
    armed = (H >= arm_threshold) if use_arming else True

    # 4) 경고 카운트
    if armed and D >= delta:
        breach_count += 1
    else:
        breach_count = 0

    # 5) 최종 상태
    if breach_count >= persistence_days and warmup_ok(t):
        alert_state = "TRIGGERED"
    elif armed and D >= 0.8 * delta:
        alert_state = "WATCH"
    else:
        alert_state = "NONE"

    # 6) 거래 이벤트 처리
    if trade_event(t):
        handle_trade_rebase_or_lot_split()

    save_state(t, E, H, D, breach_count, alert_state)
```

## 5.4 거래 이벤트 처리 권장
- 기본: **Lot 분리 추적 + 포지션 가중 합산**
- 단순버전(초기 출시): 추가매수 시 리베이스 + 로그에 "이력 단절" 명시

## 7) 경고 문구/UX 권장

## 7.1 경고 레벨
- **WATCH(주의)**: \(D_t\ge 8\%p\)
- **ALERT(경고)**: \(D_t\ge10\%p\) + 지속조건 충족
- **HIGH ALERT(강경고)**: ALERT + 절대손실/추세악화 동시

## 7.2 메시지 템플릿
- 제목: `상대성과 경고: [종목명]`
- 본문 예시:
  - `최근 벤치마크 대비 초과수익률 고점 대비 하락폭이 11.2%p입니다.`
  - `현재 E_t: -3.4%p / 고점 H_t: +7.8%p / 조건: 3일 연속 충족`
  - `의미: 절대손실 여부와 별개로 알파 반납이 진행 중입니다.`
  - `권장 행동: 비중 축소 검토, 재진입 기준 설정(예: E 회복/추세 회복)`

## 7.3 UX 원칙
1. 수식값을 그대로 노출(E,H,D,임계,지속일)
2. "왜 떴는지" 한 문장 설명(설명가능성)
3. 스누즈/재알림 주기 제공(알림 피로 감소)
4. 추가매수로 기준이 바뀐 경우 배지 표시("기준 재설정")
5. 데이터 품질 이슈(결측/조정지연) 동시 표기

---

## 8) 최종 권장 운영안(실전 기본값)

- 계산: **TR 기준 누적수익률 차이**(동일 통화)
- 룰: \(D_t=H_t-E_t\ge10\%p\)
- 보완 1: arming \(H_t\ge+5\%p\) + 보조룰 \(E_t\le-10\%p\)
- 보완 2: 3거래일 지속 확인
- 보완 3: 절대손실(예: -8%) 또는 추세(120/200MA) 결합
- 보완 4: 변동성 동적 임계(6~15%p 클램프)

---

## 9) 결론(5줄)
1. 기본 룰은 수학적으로 일관되며, 본질적으로 **상대성과(알파) 드로우다운 감지기**다.
2. 절대손익과 분리된 신호이므로, "손절"이 아니라 "벤치마크 대비 경쟁력 약화" 경고로 해석해야 한다.
3. 실전 성능은 데이터 정의(TR/통화/캘린더)와 추가매수 처리 방식(lot vs 리베이스)에 크게 좌우된다.
4. 오탐/미탐을 줄이려면 arming threshold, 지속일수, 절대손실·추세·실적 결합이 사실상 필수다.
5. 초기 배포는 고정 10%p + 3일 확인 + 데이터 품질 게이트로 시작하고, 이후 변동성 기반 동적 임계로 고도화하는 것이 안전하다.
