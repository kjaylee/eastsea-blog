  GitHub Enterprise Cloud(EMU)에서 이제 사용자 네임스페이스 리포지토리까지 엔터프라이즈 IP 허용 목록 정책을 강제할 수 있게 됐습니다.

  이는 보안팀 관점에서 예외 경로를 줄여 감사 난도를 낮추는 변화라서, 대규모 조직일수록 네트워크 경계 정책의 실효성이 크게 올라갑니다.
  → 링크: https://github.blog/changelog/2026-02-23-ip-allow-list-coverage-extended-to-emu-namespaces-in-public-preview

- **[GitHub 조직 API, Enterprise Team/Organization Team 통합 조회 지원]** (GitHub Changelog)
  GitHub는 기존 조직 API 엔드포인트에서 엔터프라이즈 팀과 조직 팀을 단일 흐름으로 조회·입력할 수 있도록 확장했습니다.
  팀 목록·멤버·리포지토리 룰셋 관련 API에서 팀 유형 필터와 구분 가능한 포맷을 제공해, 이전처럼 권한별로 분리 조회하던 복잡도를 줄였습니다.
  결과적으로 권한 자동화 스크립트가 단순해지고, 권한 누락/중복 같은 운영 사고를 사전에 줄일 수 있다는 점이 개발조직에 직접적인 이익입니다.
  → 링크: https://github.blog/changelog/2026-02-23-enterprise-team-support-in-organization-apis
