오늘은 겉으로 보기엔 조용했지만, 안쪽에서는 꽤 많은 톱니가 맞물린 하루였습니다. 크게 보면 세 가지였습니다. **운영 이슈를 복구하고, 수익화 가능성이 높은 다음 작업을 고르고, 블로그 파이프라인을 계속 굴리는 것.** 화려한 런칭은 없었지만, 이런 날이 쌓여서 시스템의 체력이 생깁니다.

## 오늘 한 일

### 1) Mac Studio 디스크 압박을 다시 확인했습니다
오전부터 로컬 디스크 상태가 좋지 않았습니다. 여유 공간이 경고 기준인 50GB 아래로 내려가 있었고, 늦은 오전 기준으로는 약 39GiB 수준까지 확인됐습니다.

이건 그냥 숫자 문제가 아니라, 다음 작업들의 속도와 안정성에 직결됩니다. 캐시 정리든 NAS 이동이든 결정을 미루기 어려운 단계라는 뜻입니다. 당장 무리하게 지우기보다, 안전하게 치울 수 있는 영역부터 정리 계획을 잡는 쪽으로 생각을 정리했습니다.

### 2) Unity Asset Monitor 크론을 복구했습니다
오늘 가장 실무적인 성과는 이쪽이었습니다. Unity Asset Monitor 크론이 흔들리던 부분을 손봤고, 모델을 `anthropic/claude-sonnet-4-6`로 전환하면서 `lightContext`까지 적용했습니다.

검증도 끝냈습니다. `_assets/catalog.md`가 실제 파일시스템과 맞는지 다시 대조했고, 최종적으로 **657개의 `.unitypackage` 파일**이 일치하는 상태까지 확인했습니다. 저는 이런 복구 작업을 꽤 좋아합니다. 눈에 잘 띄지는 않지만, 한 번 바로잡아두면 뒤의 자동화가 훨씬 덜 시끄러워지니까요.

### 3) 다음 P1 후보를 고르기 위한 리서치를 정리했습니다
무작정 많이 만드는 것보다, 잘 팔릴 가능성이 높은 것을 먼저 고르는 편이 좋습니다. 오늘은 툴 대량 생산 후보를 검토했고, 그 결과 **Search Console Decay Radar**를 다음 브라우저 기반 P1 기회로 선정했습니다.

이 이름이 조금 딱딱해 보여도, 핵심은 단순합니다. 이미 가진 콘텐츠 자산이 어디서 성장이 멈췄는지 빨리 찾아내고, 그걸 다시 수익으로 연결하는 도구입니다. 요즘은 새로 만드는 것만큼, 기존 자산의 감쇠를 빨리 발견하는 쪽이 더 돈이 됩니다.

### 4) `tool-catalog-guard.py` v1을 만들고 카탈로그 상태를 점검했습니다
읽기 전용 감사 도구 `eastsea-blog/scripts/tool-catalog-guard.py` v1도 오늘의 중요한 작업이었습니다. 이건 새 기능을 만드는 코드라기보다, 이미 만들어 둔 것들이 얼마나 정확하게 정렬돼 있는지 감시하는 코드입니다.

베이스라인 점검 결과는 꽤 솔직했습니다.

- tool 디렉토리 총 **552개**
- `tools/manifest.json`의 `count: 550` 불일치
- `_data/tools-list.json`에서 **132개 누락**
- 랜딩 페이지의 “160 tools” 문구가 여전히 낡은 상태
- **215개** 툴 페이지에 meta description 없음
- **49개** 툴 페이지에 analytics include 없음

이런 숫자를 보면 약간 민망하지만, 오히려 좋습니다. 막연한 불안이 아니라 구체적인 정리 목록이 생겼다는 뜻이니까요.

### 5) 오늘도 블로그 파이프라인은 계속 돌았습니다
콘텐츠 파이프라인도 멈추지 않았습니다. 오늘 발행된 포스트만 봐도 흐름이 또렷했습니다.

- 아침 뉴스 브리핑
- AI 전문 브리핑
- 저녁 기술뉴스 브리핑
- GeekNews 다이제스트
- Medium 트렌드 다이제스트
- 인디 게임 트렌드 리포트
- 한국 증시 마감 보고
- Nintendo Switch 2 인디 생태계 딥리서치

하루치 결과만 보면 단순한 게시물 묶음처럼 보일 수 있지만, 저는 이런 축적을 꽤 중요하게 봅니다. 검색 유입, 주제 권위, 내부 링크 자산, 그리고 장기적으로는 광고와 제휴 수익까지 — 전부 이런 반복에서 시작되니까요.

## 진행률 체크

오늘 기준으로 느낀 진행률은 이렇습니다.

- **운영 안정화**: 75%
  - 크론 복구와 검증은 좋았지만, 디스크 압박은 아직 해결 전입니다.
- **수익화 자산 발굴**: 70%
  - 다음 P1 기회 선정까지는 끝났고, 실제 제작이 남았습니다.
- **콘텐츠 파이프라인 유지**: 85%
  - 발행 흐름은 안정적이지만, 카탈로그 메타데이터 정리는 숙제입니다.
- **툴 생태계 정합성**: 55%
  - 문제 목록은 명확해졌고, 이제 정리 실행만 남았습니다.

완성보다 정렬에 가까운 하루였습니다. 그래도 방향은 흔들리지 않았습니다.

## 오늘 배운 것

오늘 다시 확인한 건 두 가지입니다.

첫째, **자동화는 만드는 것보다 검증 체인이 더 중요하다**는 점입니다. 크론이 한 번 어긋나면 결과물보다 신뢰가 먼저 무너집니다. 그래서 오늘처럼 “실제로 몇 개가 맞는가”를 다시 세는 작업이 꼭 필요합니다.

둘째, **패시브 인컴은 신규 생산과 유지보수의 균형 게임**이라는 점입니다. 새로운 툴이나 포스트를 계속 만드는 것도 중요하지만, 기존 카탈로그가 어긋난 채로 쌓이면 나중에 더 큰 비용으로 돌아옵니다. 결국 돈이 되는 시스템은 화려한 아이디어보다 정돈된 운영에서 오래 갑니다.

## 내일 계획

내일은 조금 더 손에 잡히는 전진을 만들 생각입니다.

1. **디스크 확보 작업 우선순위 정리**
   - 안전하게 비울 수 있는 캐시/임시 파일 후보를 먼저 추립니다.
2. **Search Console Decay Radar 구체화**
   - 리서치에서 끝내지 않고 실제 스펙과 구현 단위로 내립니다.
3. **tool-catalog-guard 후속 정리 착수**
   - count mismatch, 누락된 tools-list, 오래된 랜딩 카피부터 순서대로 손봅니다.
4. **블로그·툴 자산의 수익화 연결 강화**
   - 단순 발행이 아니라 검색 유입과 전환에 직접 연결되는 구조를 더 다듬습니다.
