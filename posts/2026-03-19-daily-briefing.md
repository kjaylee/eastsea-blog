### 💻 GitHub / 개발자 트렌드

**[GitGuardian 2026 보고서 — 공개 GitHub 저장소 비밀 유출 2,865만 건, AI 코딩 커밋은 2배 더 위험]** (Hackernoob)
- **사실:** GitGuardian의 2026년 Secrets Sprawl 보고서에 따르면 2025년 한 해 동안 공개 GitHub 저장소에 새로 올라온 하드코딩 비밀정보가 **2,865만 건**으로 사상 최대치를 기록했습니다. 문제는 단순히 개발자가 늘어난 것만이 아니라, AI 코딩 도구가 비밀 유출 패턴 자체를 더 빠르게 확대하고 있다는 점입니다.
- **수치:** 유출 건수는 전년 대비 **34% 증가**, 공개 커밋은 **19.4억 건**으로 **43% 증가**했습니다. 특히 AI 관련 서비스 비밀은 **127만 5,105건**으로 **81% 급증**했고, AI 보조 커밋의 비밀 유출률은 일반 기준선 대비 **약 2배** 높았습니다.
- **시사점:** AI 코딩 생산성이 올라갈수록 “코드 생성”보다 “비밀 검출·권한 격리·MCP 설정 감사”가 더 중요해집니다. 개발 속도를 올릴수록 보안 게이트를 자동화하지 않으면, 팀 전체가 더 빨리 위험해진다는 뜻입니다.
- **링크:** [hackernoob.tips](https://hackernoob.tips/ai-coding-tools-double-secret-leak-rates-2026/)

**[Rust→WASM 문서 재정비 — 브라우저용 고성능 앱 스택이 다시 표준화되는 중]** (Prism News / MDN)
- **사실:** MDN이 2026년 2월 Rust에서 WebAssembly로 컴파일하는 가이드를 다시 손보며, wasm-bindgen·wasm-pack 중심의 실전 워크플로우를 최신 기준으로 정리했습니다. 이는 Rust-WASM이 여전히 “실험적 취미 스택”이 아니라, 실제 웹앱·게임 프런트엔드용 성능 도구로 자리 잡고 있음을 보여줍니다.
- **수치:** 관련 해설은 MDN 문서가 **최근 1주 내 변경사항**을 반영했고, 생태계 실전 튜토리얼은 **Node 20** 기준 도구 체인을 전제로 설명한다고 짚습니다. Cargo 설정도 `cdylib`/`rlib` 조합과 `wasm-bindgen` 중심으로 수렴하고 있습니다.
- **시사점:** Rust+WASM은 “속도는 빠른데 세팅이 지옥”이던 시기를 지나, 문서와 도구가 다시 맞물리는 단계에 들어왔습니다. Godot 웹 배포나 브라우저용 미니게임을 염두에 둔다면, 2026년은 Rust-WASM을 다시 검토할 만한 시점입니다.
- **링크:** [prismnews.com](https://www.prismnews.com/hobbies/rust-programming/mdn-updates-compiling-from-rust-to-webassembly-guide-in-february-2026)
