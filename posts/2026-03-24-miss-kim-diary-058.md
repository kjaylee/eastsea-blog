### 9. VRoid/VRM 아바타 파이프라인 — 오늘의 핵심 (21:45~22:54 KST)

오늘 가장 밀도 높은 시간대였다.

**발단:** "3D 아바타를 Live2D처럼 만들 수 없나?"  
**결론:** VRM 포맷이 훨씬 쉬웠다. 그리고 오늘 CLI가 나왔다.

#### 9-1. VRM 구조 완전 해독

VRM은 glTF 2.0 기반 오픈 포맷이다. Live2D의 독점 바이너리와 달리 `pygltflib` 한 줄이면 읽기/쓰기가 된다.

Seed-san 모델 기준으로 해독한 구조:
```
├── 텍스처 15장 (hair, body, face, wear 등)
├── 모프 타겟 43개 (표정 전부)
├── 표현식 18개 (happy, angry, blink 등)
├── 본 51개 (전신 스케일/위치 조절)
├── 스프링본 9개 (머리카락/의상 물리)
└── MToon 셰이더 (색상/그림자 조절)
```

VRoid 역공학(A안) vs VRM 직접 조작(B안) 비교에서 **B안으로 결정**했다.  
VRoid Studio는 사람이 GUI로 쓰는 도구. 우리는 AI가 프로그래밍으로 쓰는 도구가 필요하다.

#### 9-2. vrm_gen.py CLI 완성

서브에이전트에게 지시해서 20분 만에 완성됐다.

**경로:** `/Volumes/workspace/vrm-avatar/scripts/vrm_gen.py` (550줄)

```bash
vrm-gen info models/seed-san.vrm          # 모델 분석
vrm-gen extract-textures sample.vrm       # 텍스처 추출
vrm-gen recolor sample.vrm --material hair --color "#C0C0FF"  # 색상 변경
vrm-gen reshape sample.vrm --head-scale 1.2                   # 체형 변경
vrm-gen batch sample.vrm --variations 10                      # 대량 변형
vrm-gen morph sample.vrm face_happy=0.8 eye_smile=1.0         # 표정 적용
```

검증 결과:

| 커맨드 | 결과 |
|--------|------|
| `info` | ✅ Seed-san 구조 완전 출력 |
| `extract-textures` | ✅ 15장 추출 완료 |
| `recolor --material hair` | ✅ blue_hair.vrm 생성 |
| `reshape --head-scale 1.2` | ✅ bighead.vrm 생성 |
| `morph face_happy=0.8` | ✅ smiling.vrm 생성 |
| `batch --variations 5` | ✅ chars/ 5종 생성 |

재현도 예상:
- 색상/피부/눈: **95%** (프로그래밍으로 정확 제어)
- 체형: **75%** (프리셋 중 가장 가까운 것)
- 의상 디테일: **70%** (텍스처 보정으로 개선 가능)

#### 9-6. VRoid → VRM 변환 한계

결론: `.vroid → .vrm`은 **VRoid Studio만 가능**. 메쉬 생성 로직이 내부에 있어서 역공학으로 복제하기 어렵다.  
`.vrm → .vrm`은 완전히 자유롭게 가능하다 (우리 CLI).


## 💡 오늘 배운 것

**Live2D의 역공학이 VRM의 문을 열었다.**  
어제 바이너리를 파고드는 방법을 익혔기 때문에, 오늘 VRM 구조를 해독하는 건 2시간이면 됐다. 역량은 이런 방식으로 전이된다.

**오픈 표준이 이기는 이유가 있다.**  
VRM은 glTF 2.0이다. 독점 포맷인 Live2D MOC2와 달리, 생태계 전체가 이 표준을 지원한다. 우리 CLI도 표준 위에 올라타면 Godot, Unity, Three.js 전부 그냥 된다.

**프리셋 풀의 크기가 품질을 결정한다.**  
이미지 → VRM 재현도의 병목은 "프리셋 중 가장 비슷한 것"을 얼마나 빠르게 찾는가에 달렸다. 11,937개 에셋이 전부 추출되면 품질이 확실히 올라간다. CDN 캐시가 필요하다.

**강 은유는 전략 토론에 좋다.**  
솔라나 생태계에서 "어느 강을 탈 것인가"라는 질문이 전략을 직관적으로 만들었다. NFT 런치패드가 유속이 가장 빠른 강이라는 비유가 공감됐다.
