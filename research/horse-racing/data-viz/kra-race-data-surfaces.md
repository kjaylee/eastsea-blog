# 한국마사회 공식 경주 데이터 표면 정리

## 출처

- 한국마사회 모바일 서울경마
- URL: https://m.kra.co.kr/race/seoul/main.do
- 한국마사회 모바일 제주경마
- URL: https://m.kra.co.kr/race/jeju/main.do
- 한국마사회 모바일 부산경남경마
- URL: https://m.kra.co.kr/race/bukyeong/main.do

## 핵심 메모

- 모바일 경마정보 화면은 `경마속보`, `출전정보`, `경주성적`, `출발정보`, `경주마 정보`, `기수 정보`, `조교사 정보`를 한 메뉴 체계로 제공한다.
- `서울`, `부산경남`, `제주`를 동일한 구조로 나눠 운영하므로 지역별 대시보드 설계에 그대로 매핑하기 쉽다.
- 검색 결과 기준으로 모바일 화면에는 일자별 경기 결과와 예정 경주가 함께 노출돼, 라이브 운영형 UI의 기본 정보 구조를 확인할 수 있다.

## 코퍼스 활용 포인트

- 정보지에서 `당일 편성`, `결과`, `인물/마필`, `속보`를 분리하는 IA 근거로 쓸 수 있다.
- 향후 자동 수집을 붙일 때도 `지역 -> 경기일 -> 경주` 흐름으로 크롤링 단위를 설계하면 된다.
- 데이터 시각화 문서의 추상적 예시를 실제 KRA 데이터 표면과 연결해 줄 수 있다.

## 후속 확장 아이디어

- 각 메뉴별 실제 파라미터 구조와 날짜 선택 흐름 기록
- 모바일/PC 화면 간 기능 차이 비교
- 일자별 결과를 구조화해 경주, 마필, 기수 차원 fact table 설계
