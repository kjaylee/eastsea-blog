2. 데이터 가치 체인 분석

2.1 KRA 데이터 흐름과 독점 구조

[경주 진행] → [판정·기록] → [배당 결정] → [결과 공개]
     ↑              ↑             ↑            ↓
  내부 only    내부 only     실시간 내부 翌日 경과만

핵심: KRA는 경주 직전~경주 중 실시간 데이터를 완전히 독점. 이는 내부자 거래 위험 최소화를 위한 구조로 보이지만, 시장 효율성 측면에서 구조적 비용이 발생.

2.2 배당률 데이터의 계층별 경제적 가치

계층데이터공개 시점경제적 가치
Layer 1최종 확정 배당률경주 후낮음 (결과 알려짐)
Layer 2최종 예상 배당률경주 직전~마감높음 (전략적 핵심)
Layer 3실시간 동적 배당률발매 중 (30초 갱신)가장 높음 (in-running 전)
Layer 4과거 배당률 시계열경주 후높음 (AI 모델 학습용)

한국 특수성: 한국은 in-running 베팅 금지 → 경주 중 실시간 배당률 공개가 내부자 거래 위험을 초래하지 않음. 이 점에서 30초 배당률 갱신의 투명한 공개는 세계적으로도 비교적 관대한 수준.

한계: 실시간中间 배당률(30초 갱신분) 시계열은 공개되지 않아 AI 예측 모델 학습에 활용 불가.

2.3 배당률 갱신 주기의 전략적 의미

  • 30초마다 갱신 = 하루 약 120~200회 배당률 변동 기회
  • 베팅 금액 급증 시 배당률突变(급변) 패턴 발생
  • 이 패턴 자체가 고价值的 시장 심리 데이터이나, 외부 연구자는 접근 불가

5. 정책 제안 (4가지)

제안 1: 복수의율(Par) 공개 — 단기 (唾手可得)

  • 内容: Race Index 형태의 복수의율 공식 발표 도입
  • 효과: 배당률 정상성 외부 검증 가능 → 시장 신뢰도 향상
  • 비용: 낮음 (기존 시스템 데이터 활용)
  • 비교: 일본 Race Index 모델 참고

제안 2: 사전 배당률 공개 시점 조기화 — 단기

  • 内容: 경주 10분 전 배당률 공식 발표 (현재: 경주 3분 후)
  • 효과: 사전 효율적 가격 발견 가능 → 시장 효율성 향상
  • 비용: 낮음 (배당률 갱신 주기는 이미 30초 — 공개 시점 조정만)
  • 참고: 영국·홍콩이 이미 적용 중

제안 3: 제한적 API 생태계 구축 — 중기

  • 内容: 확정 데이터 (결과·배당률·마체 정보) 유료/무료 API 공개
  • 효과: 3자 플랫폼 성장 → 데이터 경제 생태계 형성
  • 비용: 중간 (API 개발·운영 필요)
  • 참고: 홍콩 HKJC Data API 모델

제안 4: Biometric 마체 데이터 수집基盤 구축 — 장기

  • 内容: 주요 마필 심박·운동량 센서 도입 및 데이터 저장
  • 효과: 동물 복지 모니터링 + 경기력 예측 모델 고도화
  • 비용: 높음 (센서 인프라·데이터베이스 구축)
  • 참고: 영국 주요 마필 대상 사례

6. 결론

한국 경마 데이터 경제는 시장 규모(6조 원) 대비 데이터 공개 수준이 현저히 낮다는 것이 핵심 발견.

우선순위제안기간기대 효과
1순위복수의율 공개단기시장 신뢰·투명성
2순위사전 배당률 조기 공개단기시장 효율성
3순위API 생태계 구축중기데이터 경제 생태계
4순위Biometric 인프라장기동물 복지·예측 고도화

4조 원 이상의 베팅 시장에서 데이터 공개 수준의 조그만 개선이 수천억 원의 시장 효율성 개선으로 이어질 수 있으며, 이는 규제 개혁의 경제적 정당성이 충분하다는 결론.
