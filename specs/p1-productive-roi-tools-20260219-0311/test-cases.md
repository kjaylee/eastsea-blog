# Test Cases — P1 Productive ROI Tools Trio

## A. Employee Training ROI Calculator

1. **정상 시나리오**
   - 입력: 직원 40명, 교육 12h, 시급 35,000, 직원당 연매출 120,000,000, 생산성 4.5%, 실현율 75%, 프로그램비 24,000,000, 연운영비 6,000,000
   - 기대: ROI/회수기간/손익분기 생산성 향상률이 모두 숫자로 표시

2. **오류 시나리오 (전형)**
   - 입력: 직원 수 0 또는 음수
   - 기대: 검증 에러 메시지 + KPI 영역 `-` 표시

3. **경계 시나리오**
   - 입력: 생산성 향상률 0.1%, 실현율 100%
   - 기대: 매우 낮은 ROI라도 계산이 정상 수행

## B. Lead Response Time ROI Calculator

1. **정상 시나리오**
   - 입력: 월 리드 1200, 현재 90분 → 목표 20분, 현재 전환율 8%, 10분 단축당 3%, 평균 계약 1,600,000, 이익률 45%, 초기 18,000,000, 월운영 1,500,000
   - 기대: 개선 후 전환율 > 현재 전환율, 추가 수주/추가 이익/ROI 표시

2. **오류 시나리오 (논리)**
   - 입력: 목표 응답시간 >= 현재 응답시간
   - 기대: "목표 응답시간은 현재보다 작아야" 오류

3. **경계 시나리오**
   - 입력: 전환율 개선율이 매우 커서 100% 초과 가능
   - 기대: 개선 후 전환율은 100%로 캡 처리

## C. Customer Support AI ROI Calculator

1. **정상 시나리오**
   - 입력: 월티켓 9000, 사람단가 12000, AI단가 1800, AI처리율 42%, 월플랫폼 15,000,000, 초기 28,000,000, 월추가효과 6,000,000
   - 기대: 월 순효익, 연 순효익, ROI, 손익분기 AI처리비율 계산

2. **오류 시나리오 (전형)**
   - 입력: AI 처리 단가 >= 사람 처리 단가
   - 기대: validation 오류(절감모델 성립 불가)

3. **경계 시나리오**
   - 입력: AI 처리비율 0%
   - 기대: 절감 없이 플랫폼 비용 반영된 결과 계산

## D. Cross-tool Integration

1. 각 페이지에 `Back to Portal` 링크 `href='/'` 존재
2. 모든 도구는 단일 `index.html`에 inline CSS/JS 포함
3. `tools/manifest.json` 신규 3 URL 포함
4. `_data/tools-list.json` 신규 3 URL 포함
5. 로컬 정적 서버에서 신규 3 URL HTTP 200
