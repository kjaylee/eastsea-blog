⚡ Quick Presets
🎛 Cache-Control Directives
📍 Visibility
public
Response may be stored by any cache (browsers, CDN, proxies).
private
Only the browser may cache. CDNs and shared proxies must not store.
⏱ Freshness
max-age
Maximum time (seconds) the response is considered fresh in the browser cache.
= 1 hour
s-maxage
Overrides max-age for shared caches (CDNs). Ignored by browsers.
= 1 day
stale-while-revalidate
Serve stale content while revalidating in the background for N seconds after expiry.
= 1 min
stale-if-error
Serve stale content if the origin returns an error (5xx) for N seconds.
= 1 day
🔄 Revalidation
no-cache
Cache stores the response but must revalidate with the server before each use. ≠ "no-cache" is not "don't cache".
must-revalidate
Cache must not serve stale content; must revalidate or return 504 when offline.
proxy-revalidate
Like must-revalidate but applies to shared caches (CDNs) only.
immutable
Content will never change; cache skips revalidation requests during max-age. Use with content-hashed assets.
🚫 Storage Control
no-store
Response must not be stored in any cache — ever. Use for sensitive data.
no-transform
Proxies/CDNs must not modify the response (e.g. no image compression).
